# Outbox is a conditional reliable handoff

Outbox is not introduced unless Tactical Design accepts durable state-and-publication handoff. Once accepted, Go code uses `github.com/go-jimu/components/ddd/message/outbox`: the producing Application constructs its own Integration Message and records it through `outbox.Recorder` in the same Infrastructure-provided transaction as the Aggregate write, while Runtime runs `outbox.Relay` against `outbox.Store` and `message.Publisher`. Delivery is at-least-once, so the accepted design also states how consumers tolerate duplicate message IDs. The current component supplies primitives rather than a complete xorm solution; the service must implement and verify transaction participation, Store persistence, claim locking, retry policy, and Relay lifecycle. References must not imply that an xorm adapter, Inbox, or exactly-once delivery already exists.
