# Integration Message contracts follow semantic authority

A Published Fact Contract is owned by the producing bounded context; its Application event handler may select an internal Domain Event, translate it to that context's generated Integration Message contract, and publish it through `github.com/go-jimu/components/ddd/message.Publisher`. This is a narrow exception to the normal ban on generated types in Application and does not admit RPC, Kafka, or foreign-context types. An Asynchronous Intent Contract is owned by the receiving bounded context; the sender uses a local semantic Application port while its Infrastructure/ACL maps to the receiver-owned contract. Receiving adapters live under `transport/messagesubscriber`, validate and map one message to one local Application use case, while bounded-context assembly registers payloads and handlers and `internal/pkg/messagebus` owns Kafka and runner lifecycle. Outbox, inbox, durable idempotency, retry, and DLQ remain separate accepted design decisions rather than consequences of using Integration Messages.
