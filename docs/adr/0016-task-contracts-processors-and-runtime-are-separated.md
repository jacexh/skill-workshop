# Task contracts, processors, and Runtime are separated

Task Queue code exists only after an Accepted Design Decision and then uses `github.com/go-jimu/components/taskqueue`, `github.com/go-jimu/contrib/taskqueue/asynq`, and the Asynq provider behind Runtime. The owning bounded context keeps versioned `TaskType`, `Definition`, payload, and task construction under `application/task`; Application may use the provider-neutral `taskqueue.Enqueuer` and `SchemaRegistry` without inventing equivalent queue ports. An inbound adapter under `transport/taskprocessor` implements one `taskqueue.Processor` for one type, decodes and validates the task, and delegates once to an Application Command. The bounded-context module contributes schemas, processors, and provider-neutral periodic definitions, including the accepted semantic cron and business timezone; `internal/pkg/taskqueue` owns Asynq/Redis clients, worker and scheduler registration, leadership/replica coordination, middleware, provider policy, Fx lifecycle, and shutdown. Expected waiting uses a bounded delayed follow-up rather than provider failure retry, periodic schedulers only enqueue normal tasks, and external bounded contexts collaborate through an accepted published API or Integration Message rather than another context's internal task.
