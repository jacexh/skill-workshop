# Covered Go concerns use one mandatory stack

The Go House Style Baseline prescribes one implementation for every covered concern: Fx for dependency injection and lifecycle; ConnectRPC for RPC; Chi for HTTP routing; Buf, Protobuf, and `google.golang.org/protobuf` for contracts generated under `gen/`; `validator/v10` for Domain business-data validation; xorm plus `go-sql-driver/mysql` for MySQL; `google/uuid` UUIDv7 for UUID identity; go-jimu Domain Event, Integration Message, Outbox, Task Queue, Asynq adapter, FSM, sloghelper, and config loader components; go-jimu Kafka with franz-go restricted to Runtime; Asynq restricted to Runtime; `log/slog` for logging; and `samber/oops` for error enrichment. When distributed tracing is accepted and an observability backend is available, use OpenTelemetry Go with OTLP and `connectrpc.com/otelconnect`; Runtime owns setup and shutdown, execution logs carry `trace_id`, `span_id`, and `request_id`, async headers propagate trace context, and Domain remains telemetry-free. Conditional mechanisms become mandatory only after their applicability or Accepted Design Decision exists. An existing alternative is a House Style conflict rather than an automatic exception, and an uncovered concern returns to design/technology selection instead of allowing Codify to choose an arbitrary substitute. The Baseline names the component while the responsible Layer, Flow, or Platform Guide owns API usage and examples.
