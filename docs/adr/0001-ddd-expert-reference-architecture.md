# ddd-expert uses progressive reference knowledge leaves

The ddd-expert plugin separates phase decisions, artifact execution, artifact structure, and reusable knowledge. The four phase SKILL files own modeling/design judgment, acceptance, return routing, and write authorization. The internal maintain-artifacts skill is an in-process protocol that validates declared phase-operation compatibility, executes authorized artifact transactions, and checks model-design revisions without owning semantic decisions. Artifact templates own the canonical directory and terminal-state document structure. Explore exclusively authorizes strategic writes, Shape exclusively authorizes Tactical Design writes, and Codify/Guard may inspect only. References own DDD concepts, house rules, verified technology guidance, and concise navigation. This keeps artifact mechanics out of phase prompts without pretending that maintain-artifacts creates a trusted caller boundary or separate agent.

Guard has an explicit review topology. Its main coordinator freezes one Review Envelope, launches independent read-only workers for Design Realization and House-Style Conformance concurrently, merges their candidates by reason family, verifies the evidence, and owns final routing. Workers return a compact JSON ledger with every obligation identifier and state but attach evidence detail only to non-clear rows; they cannot delegate. Each unresolved merged family receives at most one bounded depth worker, but the coordinator never substitutes either complete required axis. An axis that still cannot complete after one retry, or a depth worker that cannot produce a terminal result in its single dispatch, leaves the Guard execution incomplete. The shared skill describes these roles and ordering without binding them to one code agent's delegation API.

The canonical common leaves are ddd-modeling, ddd-core, and ddd-collaboration, with database as the single Persistence House Style. Go is the primary implementation path: its compact baseline routes to Layer Guides for Domain, Application, Transport, and Infrastructure; Flow Guides for CQRS, events/messages, and taskqueue; and Platform Guides for scaffold and Runtime. Lower-frequency Python and TypeScript support remains in one compact language guide each, organized into focused sections rather than separate leaves. Files remain flat under references, and the Claude and Codex plugin tracks stay mirrored.

House Rules are conditional on their stated concern but mandatory once applicable. Design-changing mechanisms require an accepted decision; covered implementation concerns use the prescribed stack. Concrete code and SQL examples stay with the leaf that owns the rule, while evaluation fixtures and phase checklists stay outside references.

The former ddd-agent-contract and ddd-modeling-gates files are retired. Useful modeling probes move into ddd-modeling; SKILL navigation points directly to the smallest relevant leaf. README inventory, link checks, mirror checks, focused contract assertions, and external eval cases protect this architecture without pinning whole paragraphs or line counts.
