# External sources cross-check the reference model

`ddd-expert` is an integrated DDD + Clean Architecture design informed by multiple mainstream viewpoints, not a transcription of any single author. Current accepted plugin decisions are authoritative for House Style. Primary literature from DDD, Clean Architecture, Hexagonal Architecture, CQRS, and related schools is used to cross-check claims, expose conflicts and omissions, and distinguish team constraints from technology facts; author prestige alone does not settle a disputed design choice.

The concurrently developed `go-jimu/template` repository may be read as a Reference Implementation for intended Go architecture and realistic examples. Because it can contain uncommitted work, its package names, APIs, logging, reliability, and other details are evidence rather than authority; each detail must be confirmed in this design session before it enters House Style.
