# In-memory Domain Event dispatch is best effort

`Save -> Events.Drain() -> event.Dispatcher.DispatchAll()` is a conditional Go House Rule for same-context, post-commit follow-ups whose loss has been accepted; it is not a universal DDD rule. Application does not drain or dispatch when persistence fails, and it does not report a committed command as failed when dispatcher admission later fails. The go-jimu dispatcher accepts work for background handling, while `event.Handler` has no error result, so successful submission does not prove successful handling or recovery. Invariants remain synchronous Domain behavior. When facts require durable handoff, recovery, or atomic state-and-publication intent, Shape must obtain an Accepted Design Decision for the appropriate reliable mechanism rather than Codify treating this in-memory flow as reliable or silently adding one.
